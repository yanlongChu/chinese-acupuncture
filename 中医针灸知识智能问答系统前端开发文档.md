# 中医针灸智能知识问答系统 · 前端技术文档

## 一、项目概述

本系统是一个基于 **React + Three.js** 构建的中医针灸智能知识问答与 3D 可视化平台，旨在通过交互式三维人体模型、智能问答与经络知识库，帮助学习者与临床工作者快速定位穴位、查询功效并进行辨证配穴。

### 核心功能
| 模块 | 功能描述 |
| --- | --- |
| 总览 Dashboard | 关键数据统计、入口导航、时辰当令提示 |
| 智能问答 QASystem | 多轮对话、穴位关键词自动识别、跳转 3D 定位 |
| 3D 穴位可视化 AcupointVisualization | 人体模型展示、穴位交互、经络筛选、六面视角切换、双击聚焦、全屏模式 |
| 知识库 KnowledgeBase | 经典条文、穴位定位、辨证要点的结构化检索 |
| 时辰当令 ShichenTip | 子午流注实时提示（针灸优选 / 日常按摩 / 作息提示） |

## 四、关键实现

### 4.1 3D 模型加载与坐标映射

**文件**：`src/App.jsx`、`src/pages/AcupointVisualization.jsx`

1. 使用 `GLTFLoader` 异步加载 `acupuncture.glb`
2. 通过 `THREE.Box3.setFromObject` 计算模型包围盒，归一化到标准人体比例
3. 实时反馈 `xhr.loaded / xhr.total` 进度（整数去重避免重复 setState）
4. 将 `acupointPositions.js` 中的穴位原始坐标（原点为中心、y∈[-1.5, 1.5]）映射到模型世界包围盒：

```js
const scaleY = worldSize.y / acupointHeight;
const scaleX = worldSize.x / acupointWidth;
const scaleZ = worldSize.z / acupointDepth;
worldPos = (acupointPos - acupointCenter) * scale + worldCenter;
```

### 4.2 六面视角切换

在 `AcupointVisualization.jsx` 中预设正 / 背 / 左 / 右 / 顶 / 底 六个相机位姿，通过平滑插值动画切换：

- 正面：`(0, 0.5, 1.4)`
- 背面：`(0, 0.5, -1.4)`
- 左侧：`(-1.4, 0.5, 0)`
- 右侧：`(1.4, 0.5, 0)`
- 顶部：`(0, 1.8, 0.001)`
- 底部：`(0, -0.8, 0.001)`

### 4.3 模糊搜索与下拉列表

**文件**：`src/pages/AcupointVisualization.jsx`

- 支持按 **穴位名 / 编码 / 归经 / 拼音 / 定位描述** 模糊匹配
- 两层匹配：子串包含 + 逐字符松散匹配（`idx` 指针推进）
- 250ms 防抖，避免每击一键就触发过滤
- 搜索结果实时以下拉列表形式展示，点击即飞至对应穴位

### 4.4 穴位交互（单击 / 双击 / 点击跳转）

| 交互 | 行为 |
| --- | --- |
| 单击穴位 | 选中（红色高亮）+ 右上角穴位信息面板 |
| 双击穴位 | 按所属身体部位区域自动缩放聚焦 |
| 3D 画布双击空白 | 取最近穴位，按身体部位缩放 |
| 问答页点击穴位链接 | `onNavigateToVisualization` → 切到 3D 页 + 聚焦穴位 |

### 4.5 经络图例激活

点击左侧经络图例项 → 设置 `filterMeridian` → 3D 场景仅显示该经络穴位，选中项高亮 + ✓ 标记 + 主题色圆点。

### 4.6 全屏模式

通过 `position: fixed + inset: 0 + z-index: 9999` 实现视图级全屏，非浏览器原生全屏（兼容性更好）。进入时保存 html/body 的 `overflow` 与 canvas 原始像素尺寸，退出时完全还原，避免水平滚动条与布局错位。支持 ESC 快捷键退出。

### 4.7 问答首问修复（ref 解决闭包陈旧值）

**文件**：`src/pages/QASystem.jsx`

问题：首次提问时 `activeConversationId` 为 null，在 `setTimeout` 回调中仍为 null（闭包陈旧值），导致 AI 回复无法匹配新建对话。

方案：引入 `activeIdRef = useRef(null)`，在创建新对话时**同步**写入 ref，setTimeout 使用已捕获的 `targetId` 局部变量，保证 AI 回复总是追加到正确的对话。

### 4.8 时辰当令实时计算

**文件**：`src/pages/ShichenTip.jsx`

- `shichenData.js` 按 2 小时一时辰划分（子丑寅卯…亥）
- 页面加载与 `setInterval(1000)` 每秒计算当前时辰与当令经脉
- 切换后自动刷新"针灸优选 / 日常按摩 / 作息提示"三块内容

### 4.9 主题与设计

- antd `ConfigProvider` 传入主题 token（`colorPrimary`、`colorInfo`、`colorSuccess`、`colorWarning`、`colorError`、`borderRadius` 等）
- 全局 `bodyStyle` 设置为 `#FAFAF8` 暖白色，配合卡片阴影与边框构成简约风格
- 所有 Card 组件使用 `styles.body`（antd v5 新 API）替代已弃用的 `bodyStyle`

- Node.js 18+

```bash

npm install

npm run dev

npm run build

npm run preview
```

### 5.3 构建产物
```
dist/
├── index.html
└── assets/
    ├── index-*.css
    └── index-*.js          # 单包 JS（约 600KB gzip）
```

### 5.4 开发约定
- 使用函数式组件 + Hooks（禁止 class 组件）
- 样式优先使用 antd 组件 + 内联 style，复杂样式放到 `App.css`
- 状态更新防止重复 setState（如进度更新、当前对话 id 使用 ref 去重）
- 3D 场景相关操作统一在 `SmartSceneController` 或 `useFrame` 回调中处理
- 新增经络 / 穴位数据时需同步更新 `acupoints361.js` 与 `acupointPositions.js`

## 七、常见问题

| 问题 | 排查与解决 |
| --- | --- |
| 3D 模型加载卡住 | 检查 `public/acupuncture.glb` 是否存在、Vite 控制台是否有资源 404 |
| 进度条百分比和宽度不一致 | 已在 `App.css` 中设置 `.app-header-progress` 宽度 100%；若仍异常可调整 `.ant-progress-inner` |
| 首次提问 AI 无响应 | 已修复：使用 ref 同步追踪 `activeConversationId`，setTimeout 使用捕获的局部 `targetId` |
| 点击穴位链接未跳到 3D 页 | 检查 `App.jsx` 中 `handleNavigateToVisualization` 是否正确传递 `highlightAcupoint` |
| 全屏后水平滚动条 | 已修复：进入时保存 canvas 像素尺寸，退出时通过双层 `requestAnimationFrame` 还原，避免布局错位 |
| antd `bodyStyle` 警告 | 已全部迁移到 `styles={{ body: {...} }}` |

## 八、后续优化建议

1. **状态管理**：引入 Zustand / Redux Toolkit 管理跨页的穴位聚焦、对话历史等全局状态
2. **性能**：对经络筛选后的穴位列表使用 `React.memo` + `useMemo` 避免重复渲染
3. **网络请求**：QA 模块接入真实后端（FastAPI / Flask），替换 `aiAnswers` 本地字典
4. **单元测试**：补充 Vitest + React Testing Library，覆盖 `fuzzyMatch`、`detectRegion`、问答流程
5. **可访问性**：为视图切换按钮添加 `aria-label`，3D 画布增加键盘导航支持
6. **国际化**：补充英文文案包，通过 antd `ConfigProvider` 切换 locale
